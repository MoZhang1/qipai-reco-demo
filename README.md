# 牌流推荐策略系统

基于《棋牌游戏视频推荐策略设计》实现的可运行策略控制台。

## 在线演示

GitHub Pages：<https://mozhang1.github.io/qipai-reco-demo/>

线上版本使用浏览器端模拟数据，不依赖本地后端；每次点击“刷新模拟数据”都会生成一组新的演示指标。

## 已实现

- 内容准入：状态、授权、版权风险、质量和推荐动作过滤。
- 七路召回：游戏、人群、场景、标签、棋牌泛娱乐、分桶热门、探索。
- 粗排：游戏匹配、用户阶段、场景、内容质量、近期表现、业务目标、新鲜度和探索。
- 精排：有效播放、完播、互动、游戏回流、生命周期提升、综合匹配与惩罚项。
- 重排频控：同类型、同作者和强导流间隔控制。
- 推荐解释：展示召回来源、粗排/精排分、概率与惩罚项。
- 内容池：按现有 Excel 标签体系组织内容和准入状态。
- UGC 赛马：T0-T4 流量池及晋级、继续测试、降权、复审判断。
- 策略配置：权重、探索比例和频控参数可调。

## 本地运行

```bash
npm run build
python3 server.py
```

访问 `http://localhost:4178`。

项目推送到 `main` 分支后，GitHub Actions 会自动构建并发布 GitHub Pages。

`server.py` 同时提供静态后台页面和模拟数据接口：

- `GET /api/health`：后台服务健康检查
- `GET /api/snapshot`：实时生成推荐运营指标、漏斗、告警与决策事件

## 后续生产化接口

当前版本由本地后台接口实时生成模拟数据。接入生产环境时建议将以下模块替换为真实服务：

- 内容库与标签服务
- 用户画像服务
- 曝光、播放、互动和导流埋点
- 推荐召回与排序服务
- UGC 审核与流量池任务
- 策略配置中心与 A/B 实验平台
